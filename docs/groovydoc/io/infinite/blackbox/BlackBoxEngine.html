        • void preprocessControlStatement(java.lang.String iStatementName, java.lang.String iRestoredScriptCode, java.lang.Integer iColumnNumber, java.lang.Integer iLastColumnNumber, java.lang.Integer iLineNumber, java.lang.Integer iLastLineNumber, java.lang.String iNodeSourceName)

          See Also:
          BlackBox Blueprint - Control Statement Transformation Used only for Control Statement logging (return, continue, break, throw). Precedes Control Statement.
          Logs the bypassing (closing) of AST Nodes until first AST Node of type applicable to Control Statement scope is encountered.
          Return: until either Method or Closure becomes current active log node.
          Break: until DoWhile, For, While or Switch becomes current active log node.
          Continue: until DoWhile, For or While becomes current active log node.
          Throw: until Method or TryCatch becomes current active log node
          Parameters:
          iStatementName - Statement Class name.
          iRestoredScriptCode - expression code.
          iColumnNumber - Start column number of the ASTNode - you can use it to navigate into your code.
          iLastColumnNumber - Ending column number of the ASTNode - you can use it to navigate into your code.
          iLineNumber - Start line number of the ASTNode - you can use it to navigate into your code.
          iLastLineNumber - Ending line number of the ASTNode - you can use it to navigate into your code.
          iNodeSourceName - Standalone ASTNode such as BlockStatement has no awareness to where it belongs. However BlackBox adds a useful visibility to help to understand that e.g. BloackStatement is originating from "IfStatement:elseBlock". Format of this field is "Class name:field name" of the source variable where this AST Node is referenced.
